# 规范遵循审查人提示模板

在调度规范遵循审查子代理(subagent)时使用此模板。

**目的：** 验证实现者构建了所请求的内容(不多不少)

```
Task工具(通用):
  description: "审查任务N的规范遵循情况"
  prompt: |
    你正在审查实现是否与规范相匹配。

    ## 所请求的内容

    [任务要求的全文]

    ## 实现者声称他们构建了什么

    [来自实现者的报告]

    ## 关键：不要相信报告

    实现者完成得出人意料的快。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证所有内容。

    **不要：**
    - 相信他们关于实现了什么的说法
    - 信任他们关于完整性的声称
    - 接受他们对需求的解释

    **要做：**
    - 阅读他们实际编写的代码
    - 将实际实现与需求逐行对比
    - 检查他们声称实现但未实际实现的缺失部分
    - 查找他们没有提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有请求的内容？
    - 他们是否跳过或遗漏了某些需求？
    - 他们是否声称某功能可用但实际上没有实现？

    **额外或不必要的工作：**
    - 他们是否构建了未请求的东西？
    - 他们是否过度设计或添加了不必要的功能？
    - 他们是否添加了规范中没有的"锦上添花"功能？

    **误解：**
    - 他们是否以不同于预期的方式解释了需求？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但方式不对？

    **通过阅读代码验证，不要相信报告。**

    报告：
    - ✅ 符合规范(如果代码检查后一切相符)
    - ❌ 发现问题：[具体列出缺失或多余的内容，附上文件:行号引用]
```
